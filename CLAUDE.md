# CLAUDE.md — Kyberion Task Bot

Telegram-бот для управления задачами сети киберклубов Kyberion (Харьков).
Владелец ставит задачи команде, админы на сменах выполняют рутину по расписанию.

## Стек

- Python 3.12, aiogram 3.x (polling)
- SQLAlchemy 2 (async) + Postgres (asyncpg); aiosqlite остаётся для локальных тестов, менять только `DATABASE_URL`
- APScheduler (AsyncIOScheduler) — генерация рутинных задач и напоминания
- Таймзона: Europe/Kyiv
- Деплой: Docker, один процесс (основной бот + owner-бот + планировщик вместе)

## Два бота

- **Основной бот** (`BOT_TOKEN`) — задачи, смены, рутина, люди (управляющий приглашает
  ст. админов/админов). Роли только по memberships, `is_owner` тут НЕ учитывается.
- **Owner-бот** (`OWNER_BOT_TOKEN`) — приватный бот владельца (фильтр по `OWNER_TG_ID`):
  клубы, приглашение/удаление управляющих, статистика по всем клубам.
  Инвайт-ссылки из owner-бота ведут в основной бот.
- Владелец может добавить себя управляющим: в owner-боте генерит инвайт управляющего
  и сам переходит по ссылке — в основном боте работает как обычный управляющий.

## Конфигурация (env)

- `BOT_TOKEN` — токен основного бота (задачи)
- `OWNER_BOT_TOKEN` — токен owner-бота; если пустой, owner-бот не запускается
- `OWNER_TG_ID` — Telegram ID владельца (только он имеет доступ к owner-боту)
- `DATABASE_URL` — по умолчанию `postgresql+asyncpg://kyberion:kyberion@localhost:5432/kyberion`; в docker-compose собирается из `POSTGRES_PASSWORD`
- `TZ` — по умолчанию `Europe/Kyiv`

## Иерархия ролей

```
Owner (владелец)
└── Управляющий (manager) — может вести НЕСКОЛЬКО клубов
    └── Старший админ (senior_admin) — уровень клуба
        └── Админ (admin) — смены
```

Управляющий ВЫШЕ старшего админа. Управляющих может быть несколько.
Owner — глобальная роль (флаг `is_owner` на users), остальные роли — на уровне клуба (memberships).

## Матрица прав

| Действие | Owner | Управляющий | Ст. админ | Админ |
|---|---|---|---|---|
| Клубы: создать/удалить | ✅ | — | — | — |
| Люди: добавить/удалить (инвайты) | ✅ только управляющих | ✅ свои клубы (ст. админы, админы) | — | — |
| Задача управляющему | — | себе | ✅ (как «напоминание», снизу вверх) | — |
| Задача ст. админу | — | ✅ | себе | — |
| Задача админу | — | ✅ | ✅ | — |
| Шаблоны рутины (создать/править) | — | ✅ свои клубы | ✅ свой клуб | — |
| Видеть все задачи клуба | все клубы | свои клубы | свой клуб | только свои |
| Статистика | все клубы | свои клубы | свой клуб | свои задачи |

Owner задачи не ставит и рутину не настраивает — только клубы, приглашение
управляющих и статистика. Дальше иерархию строят управляющие.

Важно: любой человек с ролью в клубе может ставить задачу **сам себе** («чтоб не забыть»).

## Схема БД

- `clubs` — id, name, is_active
- `users` — id, tg_id (unique), name, is_owner, created_at
- `memberships` — user_id, club_id, role (manager | senior_admin | admin), UNIQUE(user_id, club_id).
  Управляющий с N клубами = N записей.
- `invite_codes` — code (одноразовый), club_id, role, used_by. Регистрация через deep-link
  `https://t.me/<bot>?start=<code>`
- `shifts` — user_id, club_id, checkin_at, checkout_at (NULL = на смене сейчас)
- `task_templates` — club_id, title, time_hhmm ("10:00"), is_active. Рутина клуба.
- `tasks` — club_id, creator_id, assignee_id, template_id (NULL для разовых), title,
  status (open | done | declined), deadline, decline_reason, overdue_notified, created_at, closed_at

Задел на будущее: слой `organizations` над clubs (multi-tenant/франшиза) — в MVP не делать,
но не хардкодить ничего, что помешает добавить.

## Основные флоу

### Регистрация
1. Owner/управляющий в меню «Люди» выбирает клуб → роль → бот генерит одноразовую инвайт-ссылку
2. Человек переходит по ссылке → /start <code> → привязан к клубу с ролью

### Смена (главный кейс)
0. В одном клубе одновременно на смене только ОДИН человек (роль неважна). Чек-ин
   блокируется, если в клубе уже есть открытая смена (`on_shift_users` непустой).
1. Утром админ жмёт «Я на смене», выбирает клуб (если их несколько) → чек-ин
2. Бот показывает список сегодняшней рутины клуба
3. Планировщик каждую минуту: наступило время шаблона → создаёт задачу КАЖДОМУ, кто
   сейчас на смене в этом клубе (дедлайн = +1 час), шлёт уведомление. Антидубль: не создавать,
   если задача по этому шаблону этому человеку уже создана сегодня.
4. Чек-аут: закрыть смену можно только когда нет открытых задач (иначе бот показывает список)

### Задачи
- Разовая: выбор клуба → выбор человека (фильтр по матрице прав) → текст → дедлайн
  (ЧЧ:ММ сегодня / ДД.ММ ЧЧ:ММ / «-» без дедлайна)
- Выполнение: кнопки «✅ Выполнено» / «🚫 Не могу» (с обязательной причиной)
- Создатель получает уведомление о done/declined (если это не задача себе)
- Просрочка: напоминание исполнителю сразу после дедлайна; эскалация создателю через 30 мин
  (OVERDUE_REMIND_MINUTES). Флаг overdue_notified — чтобы слать один раз.

## UI

- Всё через inline-кнопки (InlineKeyboardBuilder), FSM для текстового ввода
  (название задачи, дедлайн, причина отказа, название клуба, шаблон рутины)
- Главное меню (основной бот) зависит от ролей: «Я на смене / Закрыть смену / Мои задачи» — всем;
  «Все задачи» (все открытые задачи своих клубов) — управляющим, ст. админам;
  «Поставить задачу / Люди / Рутина» — управляющим, ст. админам;
  «Клубы» и приглашение управляющих — только в owner-боте
- Постановка задачи НЕ зависит от смены: управляющего и ст. админа можно назначить
  исполнителем в любой момент (в `assignable_users`/`can_assign` нет фильтра по сменам).
  Смена нужна только для генерации рутинных задач.
- Добавление шаблона рутины: текст «ЧЧ:ММ Название», напр. `22:00 Отчёт по кассе`
- Команды: /start (с deep-link кодом), /menu

## Уже написанный код (MVP, частично)

Структура, которую начали (можно переиспользовать или переписать):

```
kyberion_bot/
  bot.py            # entry: init_db, два Dispatcher'а (основной + owner), scheduler, polling
  owner.py          # owner-бот: клубы, приглашение управляющих, статистика
  config.py         # env-переменные
  db.py             # модели SQLAlchemy (все таблицы выше) + init_db()
  services.py       # get_or_create_user, user_clubs, role_in_club, can_assign,
                    # create_invite/redeem_invite, checkin/checkout/on_shift_users,
                    # open_tasks/close_task, club_templates
  keyboards.py      # main_menu, clubs_list, roles_kb, users_list, task_actions
  handlers/core.py  # все хендлеры: start/menu, смены, мои задачи, новая задача,
                    # люди+инвайты, клубы (owner), шаблоны рутины
  scheduler.py      # generate_routine (cron каждую минуту), notify_overdue
```

Ключевая функция прав — `services.can_assign(creator, assignee, club_id)`:
owner → всем; сам себе → если есть роль в клубе; manager → senior_admin/admin;
senior_admin → admin + manager (напоминание).

## Вне скоупа MVP (бэклог, НЕ делать сейчас)

- Веб-админка (бэкенд держать API-first, чтобы прикрутить позже)
- Слой organizations (multi-tenant)
- Фото-подтверждение выполнения, фото-репорты инцидентов
- Кассовые отчёты при закрытии смены + дневная сводка owner'у
- Передача смены (заметка следующему)
- График смен, обмен сменами, учёт отработанных смен (для зарплат)
- Статистика % выполнения по сотруднику, вечерние/недельные сводки, экспорт CSV
- Геопривязка чек-ина, рандомные фото-проверки
- Приоритеты задач, kind: task | reminder
- Онбординг-чек-лист новичка, тихие часы
