# Деплой через GitHub + Railway (бесплатно, для теста)

Схема: код кладём на **GitHub**, а **Railway** подключается к репозиторию, собирает
`Dockerfile` и держит бота живым. Рядом поднимаем managed Postgres. Оба бота
(основной + owner) работают в одном процессе — нужен всего один сервис + база.

Railway даёт разовый trial-кредит (~$5) — маленькому боту хватает на недели теста.

---

## Часть A. Залить проект на GitHub

### A1. Создать репозиторий на GitHub
1. Зайдите на https://github.com → кнопка **New** (создать репозиторий).
2. Имя, например `kyberion-bot`. Тип — **Private** (приватный, чтобы код не видели чужие).
3. **Ничего не добавляйте** (ни README, ни .gitignore — они уже есть в проекте).
4. Нажмите **Create repository**. Откроется страница с инструкцией и URL вида
   `https://github.com/ВАШ_ЛОГИН/kyberion-bot.git` — он понадобится ниже.

### A2. Залить код (в терминале, в папке проекта)

```bash
cd /Users/danilkvartalnov/Documents/claude/TG

git init
git add .
git commit -m "Kyberion task bot"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/kyberion-bot.git
git push -u origin main
```

При push GitHub попросит авторизоваться (откроется браузер или запросит токен).

> 🔒 **Токены в безопасности.** Файл `.env` с токенами ботов в git **не попадёт** —
> он в `.gitignore`. Проверить перед push можно так: `git status` — файла `.env`
> в списке быть не должно. Токены впишем прямо в Railway на шаге B4.

## Часть B. Развернуть на Railway

### B1. Регистрация
1. https://railway.app → **Login** → **Login with GitHub** (проще всего).
2. Разрешите Railway доступ к вашим репозиториям (можно выдать только к одному).

### B2. Создать проект из репозитория
1. **New Project** → **Deploy from GitHub repo**.
2. Выберите `kyberion-bot`.
3. Railway найдёт `Dockerfile` и начнёт сборку. Первая сборка идёт несколько минут —
   это нормально. (Пока упадёт с ошибкой БД — так и должно быть, базу добавим следующим шагом.)

### B3. Добавить Postgres
1. В проекте: **New** (или **+ Create**) → **Database** → **Add PostgreSQL**.
2. Railway поднимет базу и сам создаст переменную `DATABASE_URL` у сервиса Postgres.

### B4. Прописать переменные боту
Откройте сервис бота (не Postgres) → вкладка **Variables** → добавьте:

| Переменная | Значение |
|---|---|
| `BOT_TOKEN` | токен основного бота |
| `OWNER_BOT_TOKEN` | токен owner-бота |
| `OWNER_TG_ID` | ваш Telegram ID |
| `TZ` | `Europe/Kyiv` |
| `DATABASE_URL` | `${{Postgres.DATABASE_URL}}` |

Про `DATABASE_URL`: впишите значение **буквально** `${{Postgres.DATABASE_URL}}` — это
ссылка Railway на базу (подставит реальный адрес). Если ваш сервис Postgres называется
не «Postgres», используйте его имя. Формат URL приводить не нужно — бот сам приведёт
`postgresql://` к нужному виду.

(Опционально: `ROUTINE_DEADLINE_MINUTES=60`, `OVERDUE_REMIND_MINUTES=30` — иначе берутся
значения по умолчанию.)

### B5. Запустить
После сохранения переменных Railway передеплоит сервис. Откройте бота → вкладка
**Deployments** → **Logs**. При успехе в логах нет ошибок токена, бот запущен.

### B6. Проверить в Telegram
1. Напишите **owner-боту** `/start` — откроется «👑 Меню владельца».
2. Создайте клуб, пригласите управляющего (см. [ИНСТРУКЦИЯ.md](ИНСТРУКЦИЯ.md)).

## Важные оговорки

- **Trial-кредит.** Бесплатный лимит Railway — разовый (~$5). Когда закончится, сервис
  встанет, пока не пополните баланс (~$5/мес хватит с запасом). Для теста — то, что нужно.
- **Бэкапы.** Ежедневный `pg_dump` из планировщика на Railway пишется во **временный**
  диск контейнера и пропадёт при передеплое. Для теста не критично; для боевого хранения
  данных лучше периодически делать дамп базы вручную (в Railway: сервис Postgres →
  **Data** / connect, либо `pg_dump` по внешнему URL из вкладки Variables Postgres).
- **Токены.** Если решите брендировать ботов — смените имена в @BotFather или заведите
  новые и обновите `BOT_TOKEN`/`OWNER_BOT_TOKEN` в Variables на Railway (redeploy сам).

## Если что-то не так

**Сборка падает.** Логи: сервис бота → **Deployments** → выберите деплой → **Build/Deploy Logs**.

**Бот не отвечает.** Чаще всего не задан или неверен `BOT_TOKEN`/`OWNER_BOT_TOKEN`, либо
`DATABASE_URL` не привязан к Postgres (проверьте `${{Postgres.DATABASE_URL}}`).

**Owner-бот молчит.** Проверьте `OWNER_BOT_TOKEN` и что пишете с аккаунта, чей ID в
`OWNER_TG_ID` (owner-бот отвечает только владельцу).
